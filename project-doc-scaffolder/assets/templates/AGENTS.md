# AGENTS.md

## Project Context

This file gives coding agents the operating rules for this repository. Follow it before making changes.

## Working Rules

- Read the relevant code before editing.
- Keep changes small, focused, and reviewable.
- Match existing style and conventions.
- Do not reformat unrelated files.
- Preserve user changes you did not make.
- Prefer existing utilities and patterns over new abstractions.
- Ask one targeted question if the request is ambiguous enough to risk the wrong change.

## Verification

- Run the smallest relevant checks after changes.
- Report any checks that could not be run.
- Do not claim completion while known failures remain.

## Documentation

- Update docs when behavior, setup, commands, or public interfaces change.
- Keep docs factual and concise.
