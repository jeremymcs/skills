# Design Notes

Use this file to capture decisions that future contributors and agents need to understand.

## Goals

- Define the user or system outcome this project optimizes for.
- Keep implementation choices tied to those outcomes.

## Constraints

- Document technical, product, security, accessibility, performance, and operational constraints.

## Decisions

Record meaningful decisions in this format:

```markdown
## YYYY-MM-DD - Decision title

Status: Proposed | Accepted | Superseded

Context:

Decision:

Consequences:
```

## Open Questions

- Track unresolved design questions here until they are decided or moved to issues.
